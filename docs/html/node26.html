Setting up auto-updating

freshquik is the automatic database update tool for Quik AntiVirus. It can work in two modes:
  • interactive - on demand from command line
  • daemon - silently in the background
freshquik is advanced tool: it supports scripted updates (instead of transferring the whole CVD file at each update it only transfers the differences between the latest and the current database via a special script), database version checks through DNS, proxy servers (with authentication), digital signatures and various error scenarios. Quick test: run freshquik (as superuser) with no parameters and check the output. If everything is OK you may create the log file in /var/log (owned by quikav or another user freshquik will be running as): 
	# touch /var/log/freshquik.log
	# chmod 600 /var/log/freshquik.log
	# chown quikav /var/log/freshquik.log
Now you should edit the configuration file freshquik.conf and point the UpdateLogFile directive to the log file. Finally, to run freshquik in the daemon mode, execute: 
	# freshquik -d
The other way is to use the cron daemon. You have to add the following line to the crontab of root or quikav user: 
N * * * *	/usr/local/bin/freshquik --quiet
to check for a new database every hour. N should be a number between 3 and 57 of your choice. Please don't choose any multiple of 10, because there are already too many clients using those time slots. Proxy settings are only configurable via the configuration file and freshquik will require strict permission settings for the config file when HTTPProxyPassword is turned on. 
	HTTPProxyServer myproxyserver.com
	HTTPProxyPort 1234
	HTTPProxyUsername myusername
	HTTPProxyPassword mypass
